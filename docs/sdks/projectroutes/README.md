# ProjectRoutes

## Overview

### Available Operations

* [getRoutes](#getroutes) - Get project routing rules
* [stageRoutes](#stageroutes) - Stage routing rules
* [addRoute](#addroute) - Add a routing rule
* [deleteRoutes](#deleteroutes) - Delete routing rules
* [editRoute](#editroute) - Edit a routing rule
* [generateRoute](#generateroute) - Generate a routing rule from natural language
* [getRouteVersions](#getrouteversions) - Get routing rule version history
* [updateRouteVersions](#updaterouteversions) - Promote, restore, or discard a routing rule version

## getRoutes

Get the routing rules for a project. Supports searching by name/ID/pattern, filtering by route type, and diffing staged changes against production.

### Example Usage

<!-- UsageSnippet language="typescript" operationID="getRoutes" method="get" path="/v1/projects/{projectId}/routes" -->
```typescript
import { Vercel } from "@vercel/sdk";

const vercel = new Vercel({
  bearerToken: "<YOUR_BEARER_TOKEN_HERE>",
});

async function run() {
  const result = await vercel.projectRoutes.getRoutes({
    projectId: "<id>",
    teamId: "team_1a2b3c4d5e6f7g8h9i0j1k2l",
    slug: "my-team-url-slug",
  });

  console.log(result);
}

run();
```

## updateRouteVersions

Promote staged routing rules to production, restore a previous production version, or discard staged changes. - `promote`: Publishes the staging version to production. - `restore`: Rolls back to a previous production version. - `discard`: Removes the staging version without publishing.
